Google Foo Bar Level 1

Problem

Commander Lambda's minions are upset! They're given the worst jobs on the whole space station, and some of them are starting to complain that even those worst jobs are being allocated unfairly. If you can fix this problem, it'll prove your chops to Commander Lambda so you can get promoted!

Minion's tasks are assigned by putting their ID numbers into a list, one time for each day they'll work on that task. As shifts are planned well in advance, the lists for each task will contain up to 99 integers. When a minion is scheduled for the same task too many times, they'll complain about it until they're taken off the task completely. Some tasks are worse than others, so the number of scheduled assignments before a minion will refuse to do a task varies depending on the task. You figure you can speed things up by automating the removal of the minions who have been assigned a task too many times before they even get a chance to start complaining.

Write a function called answer(data, n) that takes in a list of less than 100 integers and a number n, and returns that same list but with all of the numbers that occur more than n times removed entirely. The returned list should retain the same ordering as the original list - you don't want to mix up those carefully planned shift rotations! For instance, if data was [5, 10, 15, 10, 7] and n was 1, answer(data, n) would return the list [5, 15, 7] because 10 occurs twice, and was thus removed from the list entirely.

Code

def answer(data, n): output = [] for i in data: count = 0 for j in data: if i == j: count += 1 if count <= n: output.append(i) return output print(answer(data, n))

Explanation

I start off by defining a method that passes two parameters, data and n. These are the inputs given by the problem. Next, I want to find all the numbers in data such that they appear ≤ n times. However, I first need to create a list to remember all of these numbers, so I initialized a list variable called output.

The next step is to iterate through each value in data. In order to find the amount of duplicates for each number, I have to create a nested for loop that iterates through data again. I also created a variable named count to keep track of the number of duplicate numbers. count increases by 1 only if the value in the first iteration (i), matches the value in the second iteration (j).

After running through the second iteration, I checked to see if count is ≤ n, because this means that the number of duplicates for that specific value in data doesn't appear more than n times. If this is true, then it gets appended to the output list created before.

Finally, after both for loops have run, I return output, which is the list of all numbers in data that appear ≤ n times.
